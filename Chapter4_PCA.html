Kickstarter is a global crowdfunding platform. Kickstarter has reportedly received more than 1.9 billion dollors in pledges from 9.4 million backers to fund 257,000 creative projects, such as films, music, stage shows, comics, journalism, video games, technology and food-related projects.

A huge variety of factors contribute to the success or failure of a project — in general, and also on Kickstarter. Some of these are able to be quantified or categorised, which allows for the construction of a model to attempt to predict whether a project will succeed or not. The aim of this project is to construct such a model and also to analyse Kickstarter project data more generally, in order to help potential project creators assess whether or not Kickstarter is a good funding option for them, and what their chances of success are.

Feature Engineering¶

  1. Project Name Features:

We suggested that how the project name is written may affect the success of the project. So we create some features related to the project name:

  • Number of Words Used
  • Number of Characters Used
  1. Project Launched Date Features:

Also, we suggested that the project first launch can affect its success. So we create some features related to the project launched date:

  • Launched Day, Week, Month,Year
  • Total Duration of the Project
  • Was project launched on weekday or weekend
  1. Project Category Features:

The category / main-category of the project may also affect its success.

In [14]:
## feature engineering
projects["num_words"] = projects["name"].apply(lambda x: len(x.split()))
projects["num_chars"] = projects["name"].apply(lambda x: len(x.replace(" ","")))
projects["launched_day"] = projects["launched"].dt.weekday
projects["launched_week"] = projects["launched"].dt.week
projects["launched_month"] = projects["launched"].dt.month
projects["launched_year"] = projects["launched"].dt.year
projects["duration"] = projects["deadline"] - projects["launched"]
projects["duration"] = projects["duration"].apply(lambda x: int(str(x).split()[0]))
projects["is_weekend"] = projects["launched_day"].apply(lambda x: 1 if x > 4 else 0)
projects["state"] = projects["state"].apply(lambda x: 1 if x == "successful" else 0)
projects = pd.concat([projects, pd.get_dummies(projects["category"]), pd.get_dummies(projects["main_category"]), pd.get_dummies(projects["launched_day"]), pd.get_dummies(projects["launched_week"]), pd.get_dummies(projects["launched_month"]), pd.get_dummies(projects["launched_year"])], axis = 1)
projects = projects.drop(["category", "main_category", "launched", "deadline", "launched_day", "launched_week", "launched_month", "launched_year"], axis = 1)

from sklearn.preprocessing import StandardScaler
from sklearn.decomposition import PCA

projects_X = projects.drop(["name", "state"], axis = 1)
scaler = StandardScaler()
projects_X = scaler.fit_transform(projects_X)

# Fitting PCA
pca = PCA()
pca.fit_transform(projects_X)
explained_var = np.cumsum(pca.explained_variance_ratio_)

# Plotting the amount of variation explained by PCA with different numbers of components
plt.plot(list(range(1, len(explained_var)+1)), explained_var)
plt.title('Amount of variation explained by PCA', fontsize = 14)
plt.xlabel('Number of components')
plt.ylabel('Explained variance')
plt.show()

print("Number of components explaining 80% of variance:", np.where(explained_var > 0.8)[0][0] + 1)
print("Number of components explaining 90% of variance:", np.where(explained_var > 0.9)[0][0] + 1)
print("Number of components explaining 99% of variance:", np.where(explained_var > 0.99)[0][0] + 1)
